创建个人博客网站的流程

创建个人博客网站的流程 首先，创建自己的网站分为动态和静态两种类型。 动态网站 动态网站是指需要服务器端运行的网站，用户访问时需要服务器端处理，生成动态页面。动态网站通常使用 PHP、Node.js、Python 等后端语言编写，可以部署在 Heroku、AWS、Azure 等云平台。 基本流程 1. 准备工作 准备好一个域名 购买一台服务器，在阿里云和腾讯云等平台购买，个人博客的话轻量级服务器就足够了 2. 服务器购买与配置 注册和登录后，完成实名认证（个人 / 企业均可） 进入「轻量应用服务器」控制台，点击「新建」 3. 服务器基础配置 进入服务器实例详情页，记录公网 IP、管理员密码（忘记可在控制台重置） 开放端口：在「防火墙」标签页，添加规则开放 80、443、8888 端口（宝塔面板必备） 重置 root 密码（可选，确保登录安全） 4. 安装与配置宝塔面板 方式 1：一键镜像安装 进入实例「应用管理」页签，找到「宝塔 Linux 面板」，点击「访问」 复制面板登录地址（如 http://公网IP:8888/随机字符串）、用户名、密码 浏览器访问登录地址，首次登录需绑定宝塔账号，按提示完成初始化 面板设置：修改默认用户名、密码、面板端口（建议改为非 8888 的常用端口，提升安全性） 方式 2：手动安装（无镜像时使用） 用 SSH 工具（如 FinalShell、Xshell）连接服务器： 主机：公网 IP，端口：22，用户名：root，密码：重置后的 root 密码 执行安装命令（以 Ubuntu 系统为例，CentOS 需换对应命令） 运行以下命令： wget -O install_panel.sh https://download.bt.cn/install/install_panel.sh && sudo bash install_panel.sh ed8484bec 输入 y 确认安装，等待 5-10 分钟，记录面板登录信息 5. 搭建博客环境与部署 宝塔面板左侧「软件商店」，搜索安装「LNMP」套件（Nginx+PHP+MySQL，适合 WordPress） 左侧「网站」→「添加站点」，填写： 域名：你的博客域名（如 xxx.com，需先解析到公网 IP） 数据库：选择 MySQL，设置用户名、密码 其他默认，提交 安装博客程序（以 WordPress 为例）： 宝塔「软件商店」搜索「WordPress」，一键安装到站点目录 访问 http://你的域名/wp-admin，按提示完成博客配置（设置标题、账号、密码） 6. 域名解析与 ICP 备案 域名解析：进入域名服务商控制台，添加 A 记录，将域名指向服务器公网 IP ICP 备案：腾讯云 / 阿里云控制台进入「备案」模块，按提示提交资料（域名、身份证、服务器信息），等待审核通过后，博客即可正常访问 7. 安全与优化（可选） 宝塔「安全」模块：开启防火墙、暴力破解防护，设置 IP 白名单 配置 HTTPS：宝塔「网站→SSL」，申请免费 Let’s Encrypt 证书，强制 HTTPS 访问 博客优化：安装缓存插件（如 WP Rocket）、优化图片，提升访问速度 常见问题与避坑指南 宝塔面板无法访问：检查防火墙是否开放 8888 端口，确认服务器公网 IP 正确 博客访问慢：选就近地域服务器，开启 CDN（腾讯云 / 阿里云免费 CDN） 环境冲突：宝塔安装前确保系统纯净，无其他 Web 环境（如 Apache、Nginx） 静态网站 静态网站是指不需要服务器端运行的网站，所有的内容都是预先写好的，用户访问时直接从服务器上下载静态页面，不需要服务器端进行处理。静态网站通常使用 HTML、CSS 和 JavaScript 等前端技术编写，可以部署在 GitHub Pages、Netlify、Vercel 等平台上，比如我就是部署在github上，在cloudflare上托管。 ...

Docker & Kubernetes 常用指令手册

1. Docker 服务管理 启动与状态 启动 Docker 服务: sudo systemctl start docker 设置开机自启: sudo systemctl enable docker.service 查看运行状态 (需 root 权限): systemctl status docker.service 查看版本信息: docker version 查看概要信息: docker info 检查配置文件: cat /etc/docker/daemon.json 查看帮助文档: docker --help 卸载 Docker 停止服务: systemctl stop docker.service ps -ef | grep docker 删除安装目录: rm -rf /etc/docker rm -rf /run/docker rm -rf /var/lib/dockershim rm -rf /var/lib/docker 查看依赖包: yum list installed | grep docker 删除依赖包: yum remove docker-common.x86_64 yum remove docker-client.x86_64 yum remove docker.x86_64 验证卸载: docker version systemctl start docker.service # 若卸载成功，上述命令应报错 2. Docker 镜像操作 查看本地镜像: docker images 查看所有镜像 (包含中间层/历史镜像): docker images -a 拉取镜像: docker pull <镜像名称>[:tag] # 示例: docker pull mysql:5.7.25 删除镜像: docker rmi <镜像名字或ID> 清理虚悬镜像 (Dangling Images): docker image prune 将容器保存为新镜像: docker commit -m "<描述信息>" -a "<作者名字>" <容器ID> <新镜像名>:<标签> # 示例: docker commit -m="add vim" -a="name" 65a4b2521717 centos-with-vim:1.0.0 注意: docker commit 是手动创建镜像的方式，可重复性弱且难以审计。生产环境建议使用 Dockerfile 构建。 ...
